# Requisitos Funcionales y No Funcionales — SmartPrice

---

## REQUISITOS FUNCIONALES

| ID   | Requisito                             | Descripción                                                                                                                                                                               | Verificación                                                                                                                                                                                                    |
|------|----------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| RF1  | Búsqueda de Productos                  | El sistema permite realizar búsquedas mediante una barra ubicada en la pantalla principal. Los resultados se presentan en tarjetas con nombre, imagen, categoría y precio, garantizando coherencia entre la consulta y los productos filtrados. | Ejecutar 5 búsquedas con términos distintos. Verificar que las tarjetas incluyan nombre, imagen y precio, y que ningún producto no relacionado aparezca en los resultados.                                      |
| RF2  | Historial de Precios                   | La vista detallada del producto integra una gráfica ilustrativa que representa la evolución de su precio durante los últimos 6 meses. La gráfica debe mantenerse estable en formato, estilo y legibilidad para todos los productos que cuenten con historial. | Revisar 5 productos con historial. Validar que la gráfica se muestre correctamente, que el intervalo de 6 meses sea visible y que el estilo visual sea uniforme.                                               |
| RF3  | Comparación de Precios                 | Al seleccionar un producto, el sistema despliega una tabla comparativa que muestra el precio por supermercado, precio por unidad (kg/lt), porcentaje de diferencia y una referencia ilustrativa de distancia. El precio más bajo debe mostrarse resaltado. | Comparar 3 productos distintos. Confirmar que aparezcan al menos tres supermercados por producto y que el indicador “mejor precio” se muestre de forma consistente.                                             |
| RF4  | Predicción de Precios con IA           | La pantalla detallada del producto incluye un módulo ilustrativo que presenta predicciones de precio para 7, 14 y 30 días, junto con un nivel de confianza visual. Esta información representa un comportamiento simulado, no cálculo real de IA. | Abrir 5 productos con módulo de predicción. Confirmar la presencia de los tres periodos y del nivel de confianza. Validar consistencia visual y ausencia de contradicciones entre productos.                    |
| RF5  | Indicador de Confiabilidad de Ofertas  | Los productos con oferta muestran una etiqueta visual con el porcentaje de descuento y un nivel ilustrativo de confiabilidad (ALTA, MEDIA, BAJA), basado en el tiempo estimado de permanencia de la oferta según el mockup. | Revisar 3 productos con descuento. Validar que la etiqueta se despliegue correctamente y que el nivel de confiabilidad se muestre en el formato establecido en el diseño.                                      |
| RF6  | Asistente de Consulta Inteligente      | El sistema integra un asistente que simula respuestas a consultas realizadas por el usuario. El asistente muestra mensajes predefinidos sobre precios, disponibilidad y recomendaciones, simulando comprensión en lenguaje natural. | Realizar 10 consultas simuladas. Verificar que cada consulta genere una respuesta predefinida coherente y que la interfaz conserve el formato conversacional esperado.                                            |
| RF7  | Gestión de Usuarios y Preferencias     | El sistema permite crear una cuenta mediante correo y contraseña, iniciar sesión y acceder a funciones personalizadas. El usuario puede gestionar favoritos, productos seguidos y listas almacenadas. | Simular 4 registros. Validar que, tras iniciar sesión, se visualicen las secciones de listas y favoritos. Confirmar que los elementos seleccionados se mantengan durante la navegación.                         |
| RF8  | Sistema de Notificaciones         | El sistema muestra alertas visuales cuando un producto seguido presenta una variación relevante: baja ≥10%, oferta activa o cumplimiento del precio objetivo. Las notificaciones se muestran en un ícono o sección específica. | Revisar 5 productos seguidos. Confirmar la aparición de alertas visuales y que cada mensaje coincida con el estado correspondiente (descuento, oferta o precio objetivo).                                        |
| RF9  | Carrito y Gestión de Listas de Compra  | El sistema dispone de una pantalla de “Carrito de Compra” donde el usuario administra su lista activa: agregar productos, modificar cantidades y eliminar artículos. El carrito actualiza el total por supermercado, el ahorro estimado y ofrece la opción de guardar la lista para uso posterior. Las listas guardadas pueden consultarse desde una sección dedicada. | Crear 3 listas de 1 a 5 productos utilizando el carrito. Validar la actualización de totales y del ahorro al agregar o quitar artículos. Confirmar el resaltado del supermercado más económico y la correcta visualización de listas guardadas. |
| RF10 | Reporte de Discrepancias          | Cada producto incorpora el botón “Reportar precio incorrecto”, el cual abre un formulario donde el usuario puede registrar la tienda, el precio observado, comentarios y adjuntar evidencia ilustrativa. | Enviar 3 reportes simulados. Verificar que el formulario muestre todos los campos requeridos y que la interfaz indique visualmente que el reporte fue registrado dentro del prototipo.                           |

## REQUISITOS NO FUNCIONALES

| Categoría                  | Descripción del requisito                                                                                                            | Experiencia proporcionada al usuario                                                                                 | Verificación                                                                                                    |
|----------------------------|----------------------------------------------------------------------------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------|------------------------------------------------------------------------------------------------------------------|
| Usabilidad                 | El sistema deberá permitir al usuario completar la búsqueda de un producto en un máximo de 4 pasos desde la pantalla principal.        | Sensación de simplicidad y control; navegación intuitiva sin esfuerzo.                                               | Prueba de observación registrando el número de pasos necesarios para obtener un resultado de búsqueda.          |
| Usabilidad                 | Todos los botones, etiquetas y textos deberán mostrar descripciones claras, directas y comprensibles sobre su función.                 | Reduce errores y permite aprender la interfaz rápidamente.                                                            | Encuesta o prueba con usuarios donde ≥ 90% identifique correctamente la función de cada elemento.               |
| Usabilidad                 | En caso de errores o búsquedas sin resultados, la interfaz deberá mostrar mensajes claros, específicos y orientados a la acción.       | Evita frustración; ayuda al usuario a comprender qué ocurrió y qué puede hacer.                                      | Evaluar mensajes mostrados; comprobar que el usuario comprende el significado sin apoyo adicional.              |
| Usabilidad                 | Cada acción del usuario (agregar al carrito, seguir un producto, marcar favorito) deberá generar retroalimentación visual inmediata.   | Sensación de control y confianza; confirma que cada acción fue registrada.                                           | Prueba funcional: verificar que cada interacción produzca un cambio visible (mensaje, icono, color).            |
| Usabilidad / Consistencia  | Los colores, iconos, tipografías y distribución de elementos deberán mantenerse uniformes en todas las pantallas del sistema.          | Experiencia fluida, familiar y predecible durante la navegación.                                                      | Revisión visual comparando pantallas clave para asegurar uniformidad.                                           |
| Usabilidad / Accesibilidad | Todos los textos, nombres de productos y precios deberán tener un nivel de contraste legible (mínimo 4.5:1 según WCAG).                | Mejora la lectura y reduce esfuerzo visual; accesible para más usuarios.                                             | Evaluar contraste mediante herramienta de verificación o pruebas visuales simples.                              |
| Usabilidad                 | Un usuario sin experiencia deberá ser capaz de realizar su primera búsqueda en menos de 2 minutos desde su primer contacto con la app. | Facilita el aprendizaje sin necesidad de tutoriales o guías.                                                          | Prueba con usuarios nuevos; el tiempo promedio de la primera búsqueda debe ser < 2 minutos.                     |
| Usabilidad / Responsividad | La interfaz deberá adaptarse correctamente a distintos tamaños de pantalla (móvil, tablet y escritorio), manteniendo legibilidad, orden y funcionalidad. | Experiencia continua y cómoda sin importar el dispositivo; evita desbordes, solapamientos y pérdida de información. | Prueba visual del mockup en tres resoluciones: <480px, 768px y >1024px, verificando que no existan cortes ni elementos superpuestos. |
| Rendimiento                | La pantalla principal deberá cargar completamente en menos de 3 segundos.                                                             | Sensación de fluidez y eficiencia al iniciar la navegación.                                                           | Cronometrar el tiempo de carga desde la apertura del prototipo.                                                 |
| Rendimiento                | Los resultados de búsqueda deberán mostrarse en menos de 5 segundos después de escribir el término.                                   | Minimiza tiempos de espera y mejora la percepción de agilidad en la aplicación.                                       | Cronometrar el tiempo entre escribir el término y visualizar los resultados.                                     |
| Rendimiento                | El chatbot deberá mostrar una respuesta en un máximo de 3 segundos tras registrar una consulta dentro del mockup.                      | Interacción dinámica, evitando interrupciones y sensación de lentitud.                                                | Cronometrar el tiempo de respuesta durante diferentes consultas simuladas.                                      |
| Usabilidad                 | El sistema deberá mostrar una estructura de navegación coherente: acceso claro a carrito, búsqueda, comparación, inicio, perfil y chat.      | Seguridad al navegar; evita que el usuario se pierda o dude sobre dónde encontrar funciones clave.                    | Revisión visual del flujo del mockup verificando que todas las secciones sean accesibles desde lugares lógicos. |
| Usabilidad                 | Las tarjetas de producto deberán mantener tamaños consistentes en texto e imagen para asegurar lectura clara y estética uniforme.      | Experiencia visual ordenada y profesional; evita esfuerzos innecesarios.                                              | Comparar 3 tarjetas y validar uniformidad en tamaños, alineación y espaciado.                                  |
